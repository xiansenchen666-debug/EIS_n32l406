2. 当前 EIS 总结

载波已经降到 31.25 kHz

由 BQ 内部 32 MHz 时钟通过 PWM_DIVISOR=0x0380 分频得到，不是 MCU 直接输出载波。

扫频结束会关闭激励

最后一个频点完成后调用 EIS_Stop()，依次关 DFE、CSADC、ADC、PWM。

唤醒前不写 EIS 寄存器

WAKE 只做 BQ 栈唤醒和初始化，避免 BQ 未醒时写寄存器导致通信异常。

项目当前代码行为说明
测试模式 单频测试、扫频测试 单频用 sin_fs；扫频用 eis_scan_freqs[] 表。
扫频点数 25 个频点 从 0.04 Hz 到 2087.29 Hz。
每个频点测试次数 TEST_TIMES_PER_FREQ = 1 每个频点测一次后进入下一个频点。
BQ EIS PWM 载波 31.25 kHz PWM_DIVISOR = 0x0380，按当前代码注释公式：32 MHz / (0x0380 + 128) = 31.25 kHz。
PWM 激励幅度 AMP1 = 0x4C = 76 按源码注释公式：AMP1 / 256 * 100%，约 29.69% 满幅。实际激励电流还取决于外部激励电路。
PWM DC 偏置 DC_SHIFT = 0x800 2048 / 4096 = 50%，正弦 PWM 中心点在中间。
扫频结束 调用 EIS_Stop() 关闭 BQ 内部 DFE、CSADC、ADC、PWM，避免结束后仍有载波和激励。
唤醒前是否 stop 否 gTaskWakeStackQuery 分支只清 MCU 状态并调用 InitBmicStack()，不再写 EIS 寄存器。

3. 上位机命令入口

上位机通过 USART2 发字符串，USART2_IRQHandler() 在收到换行 \n 或回车 \r 后解析命令。

命令触发标志后续主循环动作返回
WAKE_STACK 或 WAKE gTaskWakeStackQuery = true 清 EIS 状态，调用 InitBmicStack() 唤醒并初始化 BQ。 ACK_WAKE
BQ_RESET 或 RESET_BQ gTaskBqResetQuery = true 清 EIS 状态，调用 BQ7982X_SoftReset()。 ACK_RESET
START_EIS gTaskStartEisQuery = true 进入扫频模式：is_scanning=1，current_scan_idx=0，update_frq=1。 每个通道输出 EIS_DATA；全部完成输出 EIS_DONE。
START_SINGLE gTaskStartSingleEisQuery = true 进入单频模式：is_scanning=0，测当前 sin_fs。 输出 EIS_DATA 和 EIS_DONE。
STOP_EIS gTaskStopEisQuery = true 立即调用 EIS_Stop()，然后清所有 EIS 状态。 当前代码没有单独打印 ACK_STOP。
SET_FREQ:xxx 或 FREQ:xxx gTaskSetEisFreq = true 只更新 sin_fs；不会立即启动测试。 无直接返回。
SET_EIS_CONFIG:freq,dft_len,hpf_hz,dsr gTaskSetEisFreq = true，gEisConfigOverride=1 用于单频测试覆盖默认 DFT/HPF/DSR 参数。条件：freq>0、dft_len>0、hpf_hz>0、1<=dsr<=10。 无直接返回。

SET_FREQ:50
START_SINGLE

扫频测试典型命令：
WAKE
START_EIS

带自定义参数的单频：
WAKE
SET_EIS_CONFIG:0.1,39063,0.1,10
START_SINGLE

4. 整体调用流程

上位机发命令USART2 中断只负责解析字符串和置位标志，不直接执行耗时 BQ 操作。
主循环消费标志根据标志进入唤醒、停止、单频、扫频或设置频率分支。
配置 BQ 参数频率、DFT、HPF、DSR、PWM 幅度、载波分频都在启动前写入 BQ。
等待 DFE/DFTSysTick 按 dfe_wait_time 和 dft_wait_time 推动下一阶段。
读取/上报/停止读实部/虚部，计算阻抗，上报 EIS_DATA，结束时关闭载波。
1. 上位机命令 USART2 中断解析字符串，置位任务标志。
2. 主循环处理标志 唤醒、复位、停止、设置频率、启动单频、启动扫频。
3. 配置并启动 EIS SysTask_SetEisDFE() 计算参数，调用 EIS_Config() 和 EIS_PWM()。
4. DFE 等待 SysTick 等待 dfe_wait_time，然后触发 gTaskEisDFT。
5. Smart Freeze / DFT SysTask_StartEisDFT() 调 EIS_SMART_FREEZE() 触发 BQ DFT。
6. DFT 等待 SysTick 等待 dft_wait_time，然后触发 gTaskEisRead。
7. 读取结果 读 EIS_STAT，确认完成后读 CV/CC 实部和虚部。
8. 上报/推进/停止 打印 EIS_DATA；扫频推进下个频点；全部结束调用 EIS_Stop()。

主调用链

USART2_IRQHandler()
  ├─ START_EIS        → gTaskStartEisQuery = true
  ├─ START_SINGLE     → gTaskStartSingleEisQuery = true
  ├─ STOP_EIS         → gTaskStopEisQuery = true
  ├─ WAKE / WAKE_STACK→ gTaskWakeStackQuery = true
  ├─ SET_FREQ/FREQ    → gTaskSetEisFreq = true
  └─ SET_EIS_CONFIG   → gTaskSetEisFreq = true + gEisConfigOverride = 1

main()
  ├─ gTaskWakeStackQuery        → InitBmicStack()
  ├─ gTaskStopEisQuery          → EIS_Stop()
  ├─ gTaskStartSingleEisQuery   → is_scanning=0, update_frq=1
  ├─ gTaskStartEisQuery         → is_scanning=1, update_frq=1
  ├─ update_frq                 → SysTask_SetEisDFE()
  │                                ├─ EIS_Para_Calculation()
  │                                ├─ ApplyEisTableConfig()  // 扫频或自定义参数时覆盖
  │                                ├─ EIS_Config()
  │                                └─ EIS_PWM()
  ├─ gTaskEisDFT                → SysTask_StartEisDFT() → EIS_SMART_FREEZE()
  └─ gTaskEisRead               → SysTask_ReadEis()
                                   ├─ EISCompleteCheck()
                                   ├─ Read_EIS_CV()
                                   ├─ Read_EIS_CC()
                                   ├─ EIS_Z_Cal()
                                   ├─ 打印 EIS_DATA
                                   ├─ 扫频：推进 current_scan_idx 或结束
                                   └─ 结束：EIS_Stop() + EIS_DONE

5. EIS 状态机和定时

变量/状态值意义谁设置
eis_run_flag0空闲/等待启动。停止、扫频切换、初始化状态。
eis_run_flag1PWM/ADC/DFE 已启动，等待 DFE 稳定。SysTask_SetEisDFE()
eis_run_flag2Smart Freeze/DFT 已触发，等待 DFT 完成。SysTask_StartEisDFT()
eis_run_flag3到了读取时刻，但 BQ 还没完成；500 ms 后再读。SysTask_ReadEis() 中 EISCompleteCheck()==0
eis_run_flag4本次测试结果已读取并处理。SysTask_ReadEis() 末尾。

SysTick 触发规则

条件动作目的
gEisSysTickCnt >= gBQDev[0].dfe_wait_time * 1000 且 eis_run_flag == 1 gTaskEisDFT = true DFE/HPF 稳定后再触发 DFT。
gEisSysTickCnt >= gBQDev[0].dft_wait_time * 1000 且 eis_run_flag == 2 gTaskEisRead = true DFT 采样时间足够后读取结果。
gEisSysTickCnt >= 500 且 eis_run_flag == 3 gTaskEisRead = true 如果上次读时 BQ 还没完成，则 500 ms 后轮询。

7.1 默认自动计算：EIS_Para_Calculation()

参数当前公式/规则写入位置意义
hpf_corner freq > 10 ? 10 : freq 不直接写入；用于计算 HPF 系数和 DFE 等待时间。 高通滤波器截止频率。低频时跟随测试频率，高于 10 Hz 后固定 10 Hz。
eis_dsr ceil((7812.5 / freq * 5) / 32500)，最大限制 10。 EIS_DFE_CTRL26.eis_dsr = eis_dsr - 1 降采样倍率。低频为了拉长观测时间，通常需要更大 DSR。
dft_len 低频超过最大长度时按采样率/频率/5bin 计算；否则取约 31250 点。 EIS_COM_CTRL2/3/4 = dft_len - 780 DFT 计算长度。当前写寄存器前扣除了 BQ 内部 780 点固定延迟。
bin_size 7812.5 / eis_dsr / dft_len 不直接写；用于算 freq_idx。 DFT 频率分辨率。
freq_idx round(freq / bin_size) EIS_COM_CTRL5/6/7 BQ DFT/PWM 目标频率索引。实际频率只能落在 bin 上。
hpf_coefficient 1 - 2*pi*hpf_corner / (7812.5 / eis_dsr) 先换成寄存器整数，再写 EIS_DFE_CTRL5/6/7 数字高通滤波器系数。
hpf_coefficent_reg round(hpf_coefficient / 0.0000009536743) EIS_DFE_CTRL5/6/7 HPF 系数的 20 bit 左右整数化结果。
dfe_wait_time max(round(5 / hpf_corner), 2) 秒 不写 BQ；给 MCU SysTick 用。 等待 HPF/DFE 稳定。频率越低，等待越久。
dft_wait_time ceil(dft_len * eis_dsr / 7812.5) 秒 不写 BQ；给 MCU SysTick 用。 等待 BQ 采够 DFT 数据。

9.2 PWM 配置

寄存器字段/值写入值来源解释
EIS_PWM_CTRL1 0x0580 amp1 = 0x4C 固定值 激励 PWM 幅度。按源码注释公式 AMP1/256，约 29.69% 满幅。它影响外部激励电流大小，但实际电流还由硬件电路决定。
EIS_PWM_CTRL6 0x0585 phase1 = 0 固定值 Tone1 的相位偏移。当前为 0，即不加相位偏移。
EIS_PWM_CTRL11/12 0x058A/0x058B dc_shift = 0x800 dc_shifth=0b1000, dc_shiftl=0 PWM 正弦波中心偏置。0x800/4096=50%，让波形围绕中点上下摆动。
EIS_PWM_CTRL13/14 0x058C/0x058D pwm_divisor = 0x0380 固定值，当前已改为 31.25 kHz BQ PWM 载波分频。当前公式：fpwm = 32MHz / (PWM_DIVISOR + 128)，所以 32MHz/(0x380+128)=31.25kHz。之前如果是 0x0000，就是 32MHz/128=250kHz。
EIS_PWM_CTRL15 0x058E eis_pwm_mode = 0b01 固定值 先把 PWM 放到配置/单音相关模式。真正启动输出在 EIS_PWM() 中改成 0b10 并写 eis_pwm_go=1。

11. Smart Freeze / DFT 触发

DFE 启动后，MCU 等待 dfe_wait_time 秒，再调用 SysTask_StartEisDFT()，它内部调用 EIS_SMART_FREEZE()。

步骤寄存器字段/值解释

2SMART_FREEZE_CTRL1 0x0236slot_sel = slot_count - 1选择触发 slot，让 DFT 在指定时间点冻结/触发。
3SMART_FREEZE_CTRL2 0x0237sf_mode = 0b00, sf_go = 1先写一次 Smart Freeze 控制动作，类似复位/预备。

12.2 读取 CV 电压结果：Read_EIS_CV()

动作寄存器字段/值解释

电压解码公式：

VC = raw_24bit_signed
     * 4096 / 8388608
     * (1 / 0.30517)
     * (1 / vc_amp_comp)
     / dft_len_struct
     * 2

动作寄存器字段/值解释

电流解码公式：

CS = raw_24bit_signed
     * cs_direction
     * 5120 / 8388608
     * (1 / 0.3725)
     * (1 / cs_amp_comp)
     / dft_len_struct
     * 2

当前 cs_direction = -1
当前 cs_amp_comp = 5

12.4 阻抗计算：EIS_Z_Cal()

代码按复数除法计算阻抗：

V = Vreal + j*Vimag
I = Ireal + j*Iimag
Z = V / I

Zreal = (Vreal*Ireal + Vimag*Iimag) / (Ireal^2 + Iimag^2)
Zimag = (Vimag*Ireal - Vreal*Iimag) / (Ireal^2 + Iimag^2)
Zmag  = sqrt(Zreal^2 + Zimag^2)
Zpha  = atan2(Zimag, Zreal)

15. 当前代码注意点

  1. 唤醒前不能写 EIS_Stop。 当前已经修正：gTaskWakeStackQuery 分支只清 MCU 状态，然后 InitBmicStack()，不会先写 EIS/ADC/PWM 寄存器。
  2. 31.25 kHz 是通过 BQ PWM 分频设置出来的，不是 MCU 直接给 BQ 250 kHz/31.25 kHz。 MCU 写 EIS_PWM_CTRL13/14 的 PWM_DIVISOR；BQ 使用自己的 32 MHz HFO/晶振路径生成 PWM 载波。
  3. 扫频结束和单频结束会关闭载波。 SysTask_ReadEis() 在扫频最后一个频点完成后、单频结果上报后都会调用 EIS_Stop()。
  4. EIS_PWM() 中 PWM_CTRL15 固定单播写。 当前 TOTALBOARDS=1 时影响有限；后续如果恢复多板堆叠，建议把它改为使用传入的 Writetype 或明确指定目标设备。
  5. 扫频表会覆盖默认计算结果。 所以如果你改了 EIS_Para_Calculation()，扫频模式未必体现出来；扫频实际用的是 eis_scan_dft_lengths/hpf/dsrs 覆盖后的值。
